## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

**記入例：**
```
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理      ├──→ パッシブブザー
                    └─────────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】                 【Arduino UNO R3】            【出力】
IRリモコン ─────────→┐                                ┌──→ パッシブブザー（警告音）
IR受信モジュール ────→┤   信号受信・認証処理・状態管理   ├──→ RGB LED（成功/失敗表示）
                     └───────────────────────────────┼──→ サーボモーター（施錠/解錠）
                                                     └──→ 4桁7セグメント表示（入力表示）
```

補足説明（図で表現しきれない場合）：
　IR受信モジュールがリモコン信号をD2で受信し、Arduinoが入力シーケンスを保持して4桁パスコード照合を行います。認証結果に応じてサーボを解錠/施錠角へ制御し、RGB LEDとブザーで状態通知します。入力中の数字や状態は4桁7セグメントディスプレイに表示します。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（数字キー入力）──→ [入力中] ──（4桁入力完了）──→ [認証中]
  ↑                └──（取消キー）────────────┐             ├──（一致）──→ [解錠中] ──────────────────────┐
  │                                          │             └──（不一致）──→ [エラー中]                   |
  ├──────────────────────────────────────────┘                                │                        ｜
  ├────────────────────────────────────────────────────────────────（再入力）──┤                        ｜
  │                                                                           └─（3回失敗）──→ [警告中]  |
  ├──────────────────（警告時間経過: 30秒）─────────────────────────────────────────────────────────┘    ｜
  └────────────（自動施錠タイムアウト: 3分）──────────────────────────────────────────────────────────────┘
```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| 待機中 | IRリモコン入力待ち。入力バッファを空にする | LED消灯、ブザー停止、7セグは待機表示 |
| 入力中 | 数字キーを受け付け、入力シーケンスを記録する | 7セグに入力値を表示、LEDは待機色 |
| 認証中 | 入力4桁と登録パスコードを照合する | LED一時点灯、ブザー停止 |
| 解錠中 | サーボを解錠角へ移動し、施錠キー入力または一定時間後に再施錠する | 緑LED点灯、必要に応じて7セグにOPEN表示 |
| エラー中 | 不一致時に失敗回数を更新し、再入力待ちへ戻す | 赤LED点滅、ブザー短音、7セグにERR表示 |
| 警告中 | 誤入力3回後、一定時間入力受付を停止する | 赤LED点滅、ブザー断続音、7セグにLOCK表示 |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-1. データ設計（使う変数・定数の洗い出し）

> ※ プログラムで扱う「情報の種類」を先に整理します。
> ソフトウェアワークシート（AI駆動開発）でやった「データの洗い出し」と同じです。

| 情報の名前 | 何を表すか | 型（イメージ） | バイト数 | 初期値 | 備考 |
|:--|:--|:--|:--|:--|:--|
| currentState | 現在の状態（待機/入力/認証/解錠/エラー/警告） | uint8_t | 1B | 0 | loop()の状態遷移で使用 |
| inputBuffer | 入力中の4桁パスコード | char[5] | 5B | "" | 4桁+終端文字 |

| passcode | 設定済みパスコード | char[5] | 5B | "1234" | 変更機能で更新可能 |
| failedCount | 認証失敗回数 | uint8_t | 1B | 0 | 3回でロック判定 |
| lockUntilMs | ロック解除予定時刻 | unsigned long | 4B | 0 | millis()基準 |
| LOCK_DURATION_MS | ロック継続時間 | const unsigned long | 0B（SRAM） | 30000 | 30秒。Flash上の定数 |
| AUTO_LOCK_DURATION_MS | 自動施錠までの時間 | const unsigned long | 0B（SRAM） | 180000 | 3分。手動施錠を優先 |

| lastIrCode | 直近に受信したIRコード | unsigned long | 4B | 0 | 長押し重複入力の抑制 |

> [!CAUTION]
> **SRAM使用量チェック（Arduino UNO R3 の上限は 2048B）**
>
> グローバル変数の合計: 約 30 B
>
> | 合計バイト数 | 判定 |
> |:--|:--|
> | ～1000B | ✅ 余裕あり |
> | 1000～1500B | ⚠️ char配列の長さを見直す |
> | 1500B超 | 🔴 AIに相談し変数を整理すること |

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| IR信号受信監視 | 10ms程度 | **millis** | 入力の取りこぼしを防ぎ、常時受信するため |
| IR重複入力の判定窓 | 150ms程度 | **millis** | 同一IRコードの連続受信を抑制し、長押し誤入力を防ぐため |
| 4桁7セグメント表示更新 | 5ms程度 | **millis** | 入力表示を滑らかに保ち、他処理と並行するため |
| エラー時LED点滅 | 300ms | **millis** | 認証処理と同時に状態表示を継続するため |
| ロック継続時間 | 30000ms（30秒） | **millis** | failedCountが3回に達した場合の入力禁止時間を統一するため |
| ロック状態の解除判定 | 50ms程度 | **millis** | lockUntilMs 到達を監視し、時間経過で待機状態へ戻すため |
| 解錠後の自動施錠タイマー | 180000ms（3分） | **millis** | 通常は手動施錠を優先しつつ、閉め忘れ防止のフェイルセーフとして時間管理するため |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針

| 異常ケース | 発生条件 | 検知方法 | 対応方針 |
|:--|:--|:--|:--|
| IR受信の重複入力 | リモコン長押しや反射で同一コードが連続受信されるとき | lastIrCode と lastIrReceivedMs により一定時間内の同一IRコードを検出 | 2回目以降を無視し、入力バッファは1回分のみ更新 |
| パスコード誤入力 | 入力4桁が登録パスコードと不一致のとき | 照合結果が不一致 | failedCountを加算し、3回で lockUntilMs = millis() + LOCK_DURATION_MS（30秒）として警告状態へ遷移 |
| 警告中の操作 | 警告中に数字キーが押されたとき | lockUntilMs > millis() を判定 | 入力を受け付けず、LED/ブザーで警告中を通知 |
| サーボ電源不足・不安定動作 | 解錠/施錠時に電圧降下が発生するとき | サーボ動作の遅延・停止を目視確認 | 安全性確保のためサーボ制御を停止し、可能なら施錠位置へ戻してLED/ブザーで異常通知。9V1A AdapterはArduino本体給電に用い、サーボは外部5V電源 + GND共通で運用する |
| 7セグメント表示異常 | 表示が欠ける/消灯するなど表示不良のとき | 起動時表示テストまたは表示更新失敗を検出 | 表示機能のみ停止し、認証と施錠機能は継続 |
