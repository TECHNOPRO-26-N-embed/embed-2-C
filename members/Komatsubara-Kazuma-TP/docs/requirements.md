## 3. 機能要件

### 3-1. このガジェットができること（必須機能）
> ※ 「〜できる」の形で箇条書き。最低3つ書いてください。

- IRリモコンからの信号を受信できる
- 設定されたパスコードと入力シーケンスを照合できる
- サーボモーターを制御して施錠・解錠を行える

### 3-2. あったらいい機能（任意・余裕があれば）

- LEDで状態を表示できる（成功/失敗）
- ブザーで通知を行える

### 3-3. やらないこと（スコープ外）
> ※ 「〜はしない」で書いてください。AIが勝手に膨らませる機能を防ぐために重要です。

- 高度な暗号化通信は行わない
- 複数ユーザーの管理機能は実装しない

### 3-4. 非機能要件（任意）
> ※ 余裕があれば書いてください。AIはここを無視しがちです。

- **安全性**：サーボモーターの異常動作時に停止する仕組みを設ける
- **応答時間**：リモコン操作後1秒以内に反応する

---

## 4. 使用部品リスト

### 4-1. グループ共通部品（グループ全員で使うもの・1つ）

**共通部品：** Arduino UNO（ELEGOO UNO R3）

**選定理由：** プログラミングが容易で、赤外線受信モジュールやサーボモーターとの接続が簡単なため。

### 4-2. 自分が追加で使う予定の部品（ELEGOOキットより）

| 部品名 | 何のために使うか（機能との対応） |
|:---|:---|
| Remote Control | 入力信号の送信 |
| IR Receiver Module | リモコン信号の受信 |
| Servo Motor SG90 | 施錠・解錠の動作 |
| RGB LED | 状態表示（成功/失敗） |
| Passive Buzzzer | 複数回失敗 |
