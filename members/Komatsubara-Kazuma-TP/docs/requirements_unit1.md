## 3. 機能要件

### 3-1. このガジェットができること（必須機能）
> ※ 「〜できる」の形で箇条書き。最低3つ書いてください。

- IRリモコンからの信号を受信できる
- 設定された数字4桁パスコードと入力シーケンスを照合できる
- サーボモーターを制御して施錠・解錠を行える

- 誤入力3回で30秒間ロックし、警告中は入力を受け付けない
- 未定義IRコードや重複入力（150ms以内）は無視する
- パスコードは数字4桁、0-9/確定/取消/変更モードキーのみ使用


### 3-2. あったらいい機能（任意・余裕があれば）

- LEDで状態を表示できる（成功/失敗）
- ブザーで通知を行える（多重失敗）
- 入力された信号を4桁7セグメントディスプレイに表示できる
- リモコンからパスワード変更できる

### 3-3. やらないこと（スコープ外）
> ※ 「〜はしない」で書いてください。AIが勝手に膨らませる機能を防ぐために重要です。

- 高度な暗号化通信は行わない
- 複数ユーザーの管理機能は実装しない

### 3-4. 非機能要件（任意）
> ※ 余裕があれば書いてください。AIはここを無視しがちです。

- **安全性**：サーボモーターの異常動作時に停止する仕組みを設ける
- **応答時間**：リモコン操作後1秒以内に反応する

- **タイミング要件**：ロック解除30秒、自動施錠3分、IR重複入力判定150ms
- **安全設計**：異常時は施錠・LED消灯・ブザー停止など安全側に動作
- **ピン・電源要件**：D0/D1未使用、サーボは外部5V電源、GND共通、D10-D12はSPI未使用前提

## 4. 使用部品リスト

### 4-1. グループ共通部品（グループ全員で使うもの・1つ）

**共通部品：** Remote Control

**選定理由：** プログラミングが容易で、赤外線受信モジュールやサーボモーターとの接続が簡単なため。

### 4-2. 自分が追加で使う予定の部品（ELEGOOキットより）

| 部品名 | 何のために使うか（機能との対応） |
|:---|:---|
| Remote Control | 入力信号の送信 |
| IR Receiver Module | リモコン信号の受信 |
| Servo Motor SG90 | 施錠・解錠の動作 |
| RGB LED | 状態表示（成功/失敗） |
| Passive Buzzer | 複数回失敗 |
| 4 Digit 7-Segment Display | 入力信号の表示 |
