# 要件定義書 — 組込み開発実習

## 1. プロジェクト概要

### 1-1. 作品タイトル
高齢者用見守りSOS緊急通知システム

### 1-2. 一言説明
赤外線リモコンで警報を開始し、FUNC/STOPで停止できる見守り警報装置。

### 1-3. 背景・目的
一人暮らしの高齢者が緊急時にスマートフォン操作なしで周囲へ異常を知らせられる手段を提供する。

---

## 2. ユーザーと利用シーン

### 2-1. 想定ユーザー
- 主利用者: 一人暮らしの高齢者
- 補助利用者: 家族、近隣支援者

### 2-2. 利用シーン
- 自宅内（寝室、居間）で急病・転倒などの緊急時
- 手元のリモコンで即時に警報を開始し、解除キーで停止

---

## 3. 機能要件

### 3-1. 必須機能
- 待機中にIR受信があれば警報状態へ遷移できる
- 警報中は赤色LEDを高速点滅（200ms周期）できる
- 警報中はブザーを連続鳴動できる
- 待機中は赤色LEDを低速点滅（1000ms周期）できる
- 警報中に停止キー（FUNC/STOP）入力で待機状態へ戻せる

### 3-2. キー判定要件
- 停止キーは FUNC/STOP とする
- 停止判定は以下のいずれかを満たせば有効とする
- rawコード一致: cancelCode（初期値 0xFFE21D）
- command一致: 0x47

### 3-3. デバッグ機能要件
- シリアル通信 9600bps を使用する
- 起動時に SOS/CANCEL の設定値を表示する
- IR受信時に raw と command を表示する
- 状態遷移時に STANDBY/ALARM を表示する

### 3-4. 非機能要件
- 応答時間: リモコン入力から1秒以内に警報開始できる
- タイミング制御: 点滅制御は millis() を用い、常時監視性を維持する
- 安全性: 想定外状態値は待機状態へフェイルバックする

### 3-5. スコープ外
- Wi-Fi、クラウド、LINE通知などの外部通信機能
- 生体センサーによる自動異常検知
- 永続保存（EEPROM等）への設定記録

---

## 4. 使用部品

| 部品名 | 役割 |
|:--|:--|
| Arduino UNO R3 | 制御本体 |
| IR受信モジュール | 赤外線信号の受信 |
| 赤色LED | 視覚警報（待機/警報の状態表示） |
| アクティブブザー | 聴覚警報 |
| 抵抗（220Ω程度） | LED電流制限 |
| ブレッドボード、ジャンパー線 | 配線 |

---

## 5. 受け入れ条件

- 電源投入後、LEDが1秒点灯して起動確認できる
- 待機中に任意のIR入力で警報へ遷移できる
- 警報中に FUNC/STOP（cmd=0x47）で停止できる
- 警報中に停止キー以外を押した場合、警報継続できる
- 待機中/警報中のLED点滅周期が設計値に一致する
